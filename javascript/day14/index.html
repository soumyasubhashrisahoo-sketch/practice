what is call stack?

call stack is a data structure that keeps track of the function calls in a program.
it is used to manage the execution of functions and to keep track of the order in which they are called.
when a function is called, a new execution context is created and pushed onto the call stack. when the function finishes executing, its execution context is popped off the stack and control is returned to the previous context.

what expensive for loop ?

an expensive for loop is a loop that takes a long time to execute,
it can be caused by a large number of iterations or by complex operations inside the loop.
it can also be caused by a loop that is not optimized, such as a loop that is not using the correct data structure or that is not using the correct algorithm.
an expensive for loop can cause performance issues and can make the program slow.

what is global execution context?

the global execution context is the default execution context in JavaScript. it is created when the JavaScript engine starts executing a script and it contains all the global variables and functions defined in the script. it is also known as the global scope.

what is local execution context?

the local execution context is created when a function is called. it contains all the variables and functions defined within that function. it is also known as the function scope. each time a function is called, a new local execution context is created and pushed onto the call stack.

what is memory heap?

the memory heap is a region of memory where all the variables and functions are stored. it is used to store the data that is needed for the execution of the program. it is also used to store the objects and arrays that are created during the execution of the program.

what is event loop?

the event loop is a mechanism that allows JavaScript to perform non-blocking operations. it is responsible for managing the execution of asynchronous code and for ensuring that the main thread is not blocked while waiting for long-running tasks to complete. the event loop continuously checks the call stack and the task queue, and it executes any tasks that are in the task queue when the call stack is empty.
